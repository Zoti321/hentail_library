# Comic 身份锚定磁盘路径

Comic 的 `comicId` 由规范化后的资源路径派生（SHA1），而非文件内容哈希。同一磁盘路径始终对应同一 Comic；移动或重命名文件会生成新 `comicId`，Library sync 时旧记录及其用户元数据、阅读历史、Series 归属会被清除，不会自动迁移。

我们选择了路径锚定，因为在本地离线库场景下，路径即用户心目中的「这本在哪」；实现与扫描 diff 简单，且能自然区分同内容的不同副本。内容哈希虽能在移动文件后保留身份，但需要全量读盘、处理重复副本与元数据合并，成本与复杂度显著更高，与当前以 Saved path 为边界的镜像同步模型也不契合。

若未来要支持「移动文件后保留进度与元数据」，应显式引入迁移或内容指纹策略，而不是在现有路径哈希上打补丁。
